# Roadmap Task List（学生项目裁剪版）

## 使用说明

- 仅保留任务清单与拆解，不包含排期。
- 难度分级：
  - `S`：小任务，单模块可完成
  - `M`：中任务，跨 2-3 个模块
  - `L`：大任务，跨协议/执行链路
  - `XL`：架构级任务，需多子任务协同

## A. 必做任务（建议优先）

### A1. 会话/工作区恢复能力（`L`）[Aromatic finished]

- `A1-1` 定义 session/workspace 持久化数据模型（`M`）
- `A1-2` 启动恢复流程与失效数据清理（`M`）
- `A1-3` 恢复失败时的错误码与可观测日志（`S`）

完成定义：
- 重启后可恢复有效 workspace/session。
- 恢复链路可观测，失败场景有稳定错误码与日志定位信息。

当前落地状态（2026-03）：
- 已完成：workspace 录制快照持久化与恢复（`workspace.save` / `workspace.restore`），包含 tab URL（不含 tabName）与录制 steps。
- 已完成：恢复专用日志事件（`workspace.restore.start/end`）和错误码（快照缺失/恢复失败）。

### A2. 任务审计视图（`L`）

- `A2-1` 统一 TaskRun 模型（taskId/workspaceName/step timeline）（`M`）
- `A2-2` step/trace/action 日志归档与索引（`M`）
- `A2-3` 查询接口（按 taskId/状态/workspace 过滤）（`M`）

完成定义：
- 可按 taskId 查看完整执行链路和失败点。
- 可快速筛选失败任务并触发回放。

### A3. A11y 树剪枝与预处理（`L`）[Aromatic finished]

### A5. 面板与扩展可用化（`XL`） [ValeKanran 负责]

### A8. Excel/CSV 数据驱动执行（`M`）

- `A8-1` Excel/CSV 读取能力（`S`）
- `A8-2` 表格列到 `StepUnion` 参数映射（`M`）
- `A8-3` 数据校验与错误提示（缺列、类型不匹配、空值策略）（`S`）
- `A8-4` 最小端到端用例（读取表格 -> 执行 -> 结果汇总）（`S`）

完成定义：
- 可直接从本地 Excel/CSV 读取数据并驱动任务执行。
- 对常见数据错误给出可定位、可理解的提示。
